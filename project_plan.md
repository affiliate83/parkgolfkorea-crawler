# ⛳ 파크골프 코리아 (Park Golf Korea) 프로젝트 계획서

## 1. 프로젝트 개요 및 목표
- **프로젝트명**: 파크골프 코리아
- **주요 목표 1**: 현재 발생 중인 프론트엔드 버그(지역별 탭 클릭 시 목록이 나타나지 않는 현상) 완벽 해결.
- **주요 목표 2**: 기술적/콘텐츠 검색 엔진 최적화(SEO)를 통해 구글 및 네이버 검색 노출을 극대화하고 오가닉 트래픽(자연 유입)을 대폭 늘리기.

---

## 2. 현재 버그 분석 및 해결 계획 (지역 탭 필터링 오류)

**문제 상황**: 첨부해주신 스크린샷을 보면 '전체' 탭에서는 721개의 골프장 데이터가 카드 형태로 잘 불러와지지만, '수도권', '대구/경북' 등의 지역 탭을 누르면 아무것도 뜨지 않습니다. 이는 UI 화면에서 데이터를 필터링하는 자바스크립트 로직이나, 지역 카테고리(Taxonomy) 값을 연결하는 부분에 오류가 났을 확률이 높습니다.

### 🛠️ 디버깅을 위해 제가(AI) 요청하는 정보
이 문제를 정확히 고치기 위해 저에게 **아래 코드 중 하나**를 복사해서 보여주시면 됩니다.
1. **프론트엔드(UI) 코드**: 지역 탭 버튼(전체, 수도권, 부산/경남 등)과 관련된 HTML, JavaScript(또는 jQuery) 코드. (클릭 시 어떤 로직으로 카드를 숨기고 보여주는지 확인 필요)
2. **워드프레스 템플릿(PHP) 코드**: 해당 페이지를 구성하는 워드프레스 테마 파일 (예: `archive-golf.php` 또는 커스텀 페이지 템플릿).
3. **데이터 구조 정보**: 네이버 API로 가져온 골프장 데이터에 '지역' 정보를 어떻게 저장했는지. (워드프레스의 기본 '카테고리' 기능을 썼는지, 아니면 ACF 같은 '커스텀 필드(메타데이터)'에 저장했는지)

## 3. 웹 트래픽 및 검색 노출 극대화(SEO) 마스터플랜

버그가 수정되고 나면, 본격적으로 사이트를 키우기 위한 전략을 실행합니다.

### Phase 1: 기술적 SEO (검색 로봇 친화적 환경 구축)
- **네이버 & 구글 웹마스터 도구 세팅**: `sitemap.xml`과 RSS를 제출하여 파크골프장 721개의 데이터가 검색 엔진에 빠르게 색인(Index)되도록 합니다.
- **스키마 마크업(Schema Markup)**: 각 골프장 페이지에 `SportsActivityLocation` 구조화 데이터를 적용하여, 구글 검색 결과에 골프장 이름뿐만 아니라 별점, 주소, 가격 등이 화려하게 노출되도록(리치 스니펫) 설정합니다.
- **URL 구조 최적화**: 검색 엔진이 이해하기 쉽도록 주소를 체계화합니다. (예: `parkgolfk.com/경기/양평/파크골프장-이름`)

### Phase 2: 콘텐츠 SEO (롱테일 키워드 장악)
- **자동화된 메타 태그 생성**: 721개 골프장의 제목(Title)과 설명(Description)을 일괄적으로 매력적으로 최적화합니다. (예: `[대구/경북] 구지1호근린공원 파크골프장 - 요금, 예약방법, 코스 정보 총정리`)
- **정보성 콘텐츠 발행**: 단순히 골프장 목록만 제공하면 재방문율이 떨어집니다. '초보자 가이드' 탭을 활성화하여 `파크골프채 고르는 법`, `파크골프 규칙`, `2026 전국 파크골프 대회 일정` 등의 글을 정기적으로 포스팅하여 트래픽을 끌어모읍니다.

### Phase 3: 사용자 경험(UX) 및 바이럴 강화
- **지도 기반 검색 고도화**: 네이버/카카오 지도 API를 붙여 "내 주변 파크골프장 찾기" 기능을 메인으로 내세웁니다.
- **리뷰/댓글 시스템 도입**: 실제로 골프장을 다녀온 어르신들이나 동호인들이 "잔디 상태가 좋다", "주차장이 넓다" 등의 리뷰를 남길 수 있게 하여 사이트 내 체류 시간을 늘리고 최신 콘텐츠가 자동 생성되게 합니다.
